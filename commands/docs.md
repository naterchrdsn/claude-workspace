Documentation task: `$ARGUMENTS`

## Documentation Workflow

1. **Define Audience and Goal**
   - Who is this for?
   - What should they be able to do after reading?

2. **Choose Document Type**
   - How-to guide, reference, or explanation
   - Optimize for scannability and task completion

3. **Draft with Examples First**
   - Start with a working example
   - Add step-by-step instructions

4. **Add Verification Steps**
   - How to confirm it worked
   - Common pitfalls and fixes

5. **Review for Clarity**
   - Plain language, clear structure, minimal ambiguity

## Output Template

Use: `docs/templates/documentation-brief.md`

## Usage

- `/docs "write API usage guide for the payments service"`
- `/docs "improve onboarding guide for new engineers"`
